# Patterns

## Session Patterns

1. **Morning sessions** = context loading. End of day sessions = execution.
2. **New tool installs** always need HERMES_CAPABILITIES.md updated after.
3. **Docker-dependent tools** can't run in this container — always check Docker availability first.
4. **Telegram as coordination layer** — Joseph DMs on Telegram, Zo handles execution.

## Business Patterns

1. **First clients come from marketplaces** (HYRVE) before word-of-mouth.
2. **Testimonials unlock higher tiers** — do free/discounted work early to build social proof.
3. **Content + affiliate** drives organic — TikTok pipeline runs daily.

## Technical Patterns

1. **Zero API cost stack** = Ollama + browser-use + Solomon Bus. Keep it free until revenue justifies costs.
2. **Memory compounding** = every session logged to raw/ → compiled to brain/ → next session reads it.
3. **Skills compound** — once a skill is installed and working, it rarely breaks.
4. **Deterministic before LLM** — mechanical tasks (URLs, state, file I/O) → code. LLM for judgment, synthesis, writing. Prevents forgetfulness structurally.
5. **Brain-first before action** — check Solomon Vault for existing context before using external APIs or acting.
6. **Entity detection on every message** — people, companies, tools, original ideas. Route to right brain file silently, don't block response.
