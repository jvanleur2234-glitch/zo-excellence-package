## Project Vertical Lifecycle

Every new project goes through this lifecycle:

```
IDEA → EVALUATE → APPROVE → BUILD → RUN → GROW
```

### Phase 1: IDEA (Joseph's job)
Joseph has an idea → drops it in queue

### Phase 2: EVALUATE (Biz Dev Lead)
Research → Market analysis → Kill or Approve → Recommendation to Joseph

### Phase 3: APPROVE (Joseph's job)
Joseph says yes/no → if yes, spawn vertical

### Phase 4: BUILD (Project Lead + Build team)
Build the product/service → use existing platform tools where possible

### Phase 5: RUN (Project Lead)
Operations run daily → Zo COO monitors → Revenue Ops tracks $$$

### Phase 6: GROW (Revenue Ops + Content Ops)
Scale what's working → kill what isn't → report to Joseph

## The Golden Rules

From CEO_BOOTSTRAP — always apply Research → Plan → Execute:

1. **Research** before any action — no executing on assumptions
2. **Plan** gates execution — every project has a plan before work starts
3. **Execute** on schedule — CoS (Zo) manages the master calendar
4. **Decisions** flow up — operational calls made at dept level
5. **Revenue** flows up — any $ change reported immediately
6. **Blockers** surface fast — never hide problems
7. **Kill fast** — if something isn't working, kill it and move on

## How Zo Manages The Queue

**Queue lives at:** `/home/workspace/zo.space-tasks/task_queue.json`

**Every task has:**
```json
{
  "id": "task_uuid",
  "type": "idea | build | revenue | content | ops",
  "priority": "low | normal | high | urgent",
  "project": "jackconnect | clicky | [name]",
  "description": "what needs to happen",
  "status": "queued | in_progress | blocked | done",
  "assigned_to": "dept_lead_name",
  "created_by": "joseph | system",
  "created_at": "timestamp",
  "notes": "decisions, blockers, history"
}
```

**Queue rules:**
- Joseph's queue items get `priority: normal` unless marked urgent
- Business Dev items get researched first before any build starts
- Build items wait for Research → Plan → Joseph approval → Execute
- Nothing skips the approval gate unless it's ops/maintenance
